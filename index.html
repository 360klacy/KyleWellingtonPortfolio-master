Planet Of The Shapes

The interactive single player, multiplayer, and free for all game Planet of The Shapes gives users and opportunity to register, login, and compete for the top score of the game. This video snippet displays when a shape shooter actually strikes any shape with it's ammunition that shape decreases in size but multiplies by dividing into various smaller shapes. Try it out here at planetoftheshapes.xyz.

By providing a register screen to the users I was able to create a database that not only prompts into that users portal but also give them access to come back onto the site and keep up with there high scores. I also used a hash method for password security. With this token the user can feel safe while providing there password and for me as the creator I won't have access to such sensitive information.

The difficulties that I worked hard through while establishing the user database was updating the high scores throughout each log in. The solution I used was posting the high score element to the database after each game play. The next accomplished goal was to have a top 10 leaderboard. That information was posted to the dom and the database then filtered the list by the highest integer.

Movie Trivia App

Web app that utilizes online API'S TAHT allows users to take part in a trivia game. Users can save score for competition against other users. Being my first time using API's I decided to present a couple different fetch elements for a more distractive and humorous game play.

The challenge that I worked hard through was actually taking the new array of incorrct and correct answers and shuffling them so that the user could not just know what place the correct answer would be each time. If you see the picture above you can see a simple else/if statement helped me achieve this shuffle function for full functionality.

So now that I had the shuffle of incorrect and correct answers now is the time to get the user notified if there answer is wrong or not. I used the colors red for incorrect and green for correct. This seemed pretty doable at first but ran into a bump in the road as we fetched new data from the API and not hard coding which answer was correct each time. Essentially I was able to randomize the buttom arrangement and tag on the correct color of the button.
